    注意事项:

    1. 列表接口都含有 page（默认 1 ）, size（默认 10） 参数，以下接口不在列出。
    • stime 开始查询时间
    • etime 结束查询时间
    • formatTime 指定具体某一天查询
    1. 响应值 code 不为 0 时，代表出错，这时 msg 基本就是错误信息，前端基本不需要根据 code 的值，提示错误信息。
    2. {id} 选填， :id 必填

    响应结果

    1. ListResult<T>

    • 说明: 返回列表时的响应结果
    {
    "code": 0,
    "state": "success",
    "msg": "成功",
    "data": {
        "total": 71, // 总条数
        "list": [  // list 为 Array<T>,otherData 为 额外对象属性。
            T,
            T,
            T,
            ......
        ],
        "otherData": {}
    }
}

    2. OneResult<T>

    • 说明： 返回单个对象时的响应结果
    {
    "code": 0,
    "state": "success",
    "msg": "成功",
    "data": T // data 为 T 类型 json  对象

    3. OperatorResult

    • 说明: 操作成功后的响应结果(比如 添加/修改值) 只要code为0，就代表操作成功，操作失败，msg 就是错误信息。
    {
    "code": 0,
    "state": "success",
    "msg": "成功",
    "data": {} 
}

    2.2 班主任营养餐上报或修改

    • 地址: [put/post] /api/report/teacher/{id} 说明，post 和 put 都可以，只要 参数带id ，就是修改，不带id，就是新增，每天每个班主任只能上报一条。
    • 参数:
      参数名称 参数类型 是否必填
      id 修改时传递 否
      formatTime string(示例:2019-01-01) 是
      yingdao int(应到人数,根据接口 /api/info/rolestudent 自动获取) 是
      shidao int 实吃人数 是
      egg int 鸡蛋数量 是
      milk int 牛奶数量 是
    • 响应:
    {
  "code": 0,
  "data": {},
  "msg": "string",  // 如果 code 不是 0，这里就是错误信息，直接提示 msg 就好
  "state": "string"
}
